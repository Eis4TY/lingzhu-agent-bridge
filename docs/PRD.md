#PRD

# 功能
1. 将各种 AI 智能体协议转为灵珠平台协议规范的服务，支持流式响应转换。
2. 支持 openAI 格式。
3. 支持 A2A 格式。
4. 自定义协议字段绑定。

支持 docker 部署
支持管理多个绑定
接收来自灵珠协议的请求，转为其他协议发给用户配置的智能体
用户配置自己的 AI 智能体，包括 url、鉴权等信息
有一个简易的 webUI 界面，便于配置和管理，监控状态


# webUI 界面
1. 绑定管理
2. 智能体管理
3. 状态监控
4. 协议管理
1. 核心看板 (Dashboard)

智能体状态监控： 列出所有已配置 Agent 的连通性（健康检查状态）。

2. 绑定管理中心 (Binding Management)
配置列表： 支持一键开关。

智能体基础配置：
Target URL： 目标 Agent 的地址。
鉴权设置： 支持 API Key (Header/Bearer)、OAuth2 或自定义 Header。
可视化协议映射编辑器： * 左右对照布局： 左侧是灵珠协议结构，右侧是目标协议（如 OpenAI）。
映射逻辑： 支持“直连（A=B）”、“固定值（A='Fixed'）”和“脚本处理（A=js_func(B)）”。

3. 协议测试沙盒 (Debug Sandbox)
模拟请求： 在 UI 上直接输入灵珠格式的 JSON。
实时转换预览： 点击“测试”后，中间层显示转换后的 JSON，以及发给目标 Agent 后的原始响应。
链路 Trace： 打印出整个转换过程的耗时明细（协议处理耗时 vs 目标 Agent 响应耗时）。

4. 日志与追踪 (Logs & Audit)
请求流水： 记录近期的转换请求，支持按错误码、绑定 ID 筛选。
Payload 详情： 能够查看失败请求的 Body 内容（注意：建议增加“隐私模式”，可选是否对日志中的 API Key 或对话内容加密）。

UI 布局建议
对于这类“映射类”产品，推荐采用 “左导航 + 中间列表 + 右侧抽屉详情” 的设计模式。这样在配置复杂的协议绑定时，不需要频繁切换页面。

Schema 自动导入： 如果灵珠平台或 OpenAI 提供了 Swagger/OpenAPI 的文档，支持通过 URL 一键导入所有字段，免去手动输入的痛苦。

模板库： 预置好常用的“灵珠 -> OpenAI”、“灵珠 -> Claude”的绑定模板，用户只需要改下 API Key 就能跑通。
